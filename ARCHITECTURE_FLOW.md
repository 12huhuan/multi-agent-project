## 详细数据流

### 路径 A — 直接工作流模式 (6 个独立工作流)

```
+-----------------------------+
|      用户在页面输入参数         |
|  例: Listing → 产品名/品类/特性  |
+-------------+---------------+
              |
              v
+-------------+---------------+
|   POST /api/v1/{domain}/run  |
|   创建 task_id, 初始化状态     |
|   asyncio.create_task() 后台  |
+-------------+---------------+
              |
              v
+-------------+---------------+
|   LangGraph StateGraph       |
|   按序执行 N 个 Agent         | <--- MemorySaver 检查点
+-------------+---------------+       HITL interrupt_after
              |
              | 以 Listing 工作流为例:
              v
+-------------+---------------+      +------------------+
| 1. KeywordResearchAgent     | ---> | Amazon Autocomplete|
|    关键词研究 + SEO 打分      |      | API (免费热词)     |
+-------------+---------------+      +------------------+
              |
              v
+-------------+---------------+
| 2. TitleGenerationAgent     |
|    生成 3 个标题候选 + 评分    |
+-------------+---------------+
              |
              v
+-------------+---------------+
| 3. BulletPointsAgent        |
|    生成 5 条卖点 (特性+利益)   |
+-------------+---------------+
              |
              v
+-------------+---------------+
| 4. DescriptionAgent         |
|    生成 SEO 友好 HTML 描述    |
+-------------+---------------+
              |
              v
+-------------+---------------+
| 5. APlusContentAgent        |
|    设计 A+ EBC 图文模块       |
+-------------+---------------+
              |
              v
+-------------+---------------+
| 6. SEOScoringAgent          |
|    四维度评分 + 优化建议       |
+-------------+---------------+
              |
              v
+-------------+---------------+
|     HITL 中断 — 等待审核      | <--- 前端轮询 GET /{task_id}/status
+-------------+---------------+
              |
    (通过)    |    (拒绝)
              v
+-------------+---------------+
|     POST /{task_id}/approve  |
|     返回最终结果给前端          |
+-------------+---------------+
              |
              v
+-------------+---------------+
|       前端结果展示             |
|   (SEO 报告 / 标题列表 / ...)  |
+-----------------------------+
```

### 路径 B — 全自动编排流水线 (Orchestrator)

```
+---------------------------------------+
|  用户在 OrchestratorPage 输入品类        |
|  例: "Wireless Headphones"             |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|  POST /api/v1/orchestrator/run        |
|  {action: "auto", context: {...}}     |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|      OrchestratorAgent                |
|      _run_full_pipeline()             |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|  1. ContextBus.create()               |
|     创建 ProductContext (Pydantic)     |
|     生成 product_id, 初始化元数据        |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|  2. Amazon Autocomplete               |
|     获取品类真实搜索热词                 |
|     LLM 兜底: 生成卖点/品牌故事          |
+-------------------+-------------------+
                    |
        +-----------+----------+----------+----------+
        |           |          |          |          |
        v           v          v          v          v
+--------+  +--------+  +--------+  +--------+  +--------+
| Phase1 |  | Phase2 |  | Phase3 |  | Phase4 |  | Phase5 |
| 选品    |  | Listing|  | 合规   |  | 社媒   |  | 评论   |
+--------+  +--------+  +--------+  +--------+  +--------+
        |           |          |          |          |
        v           v          v          v          v
   Selection    Listing    Compliance   Social     Review
   Workflow     Workflow   Workflow    Workflow   Workflow
   (4 Agent)   (6 Agent)  (3 Agent)   (5 Agent)  (5 Agent)
        |           |          |          |          |
        +-----------+----------+----------+----------+
                    |
                    v
+-------------------+-------------------+
|  每个 Phase 的三步循环:                 |
|                                       |
|  a. bus.derive(domain, ctx)  上下文→状态 |
|  b. _execute_with_state()   运行工作流   |
|  c. bus.ingest(domain, result, ctx)   |
|     结果→上下文 (零 LLM 开销)            |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|  ContextBus.save(ctx)                 |
|  持久化到 data/contexts/{id}.json     |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|  返回 decisions[] 数组                 |
|  每个环节: status + results 摘要       |
+-------------------+-------------------+
                    |
                    v
+-------------------+-------------------+
|    前端展示完整流水线结果                |
|    (选品推荐 + Listing + 合规 + 社媒 + 评论)|
+---------------------------------------+
```

### Customer Service 流式对话 (SSE)

```
+-----------------------------+
|  用户在客服页面发送消息          |
+-------------+---------------+
              |
              v
+-------------+---------------+
|  POST /conversations/{id}    |
|       /messages/stream       |
|  Server → SSE text/event-stream|
+-------------+---------------+
              |
              v
+-------------+---------------+      +------------------+
| Phase 1: IntentRecognition  | ---> | ChromaDB 向量检索  |
| 意图分类 (10+ 种) + 情感分析  |      | (Qwen Embedding) |
+-------------+---------------+      +------------------+
              |                              |
              v                              v
+-------------+---------------+      +------------------+
| Phase 2: KnowledgeRetrieval | <--- | Top-K 知识块      |
| 取回相关知识库内容              |      +------------------+
+-------------+---------------+
              |
              v
+-------------+---------------+
| Phase 3: ReplyGeneration    |
| token-by-token 流式生成      | ---> SSE 逐字推送到前端
+-------------+---------------+
              |
              v
+-------------+---------------+
| Phase 4: EscalationDecision |
| 规则+LLM: 是否需升级/生成工单  |
+-------------+---------------+
              |
   (升级)     |     (正常)
              v
+-------------+---------------+      +-------------+
| TicketGenerationAgent       | ---> | 前端 SSE:   |
| 生成工单 (优先级/摘要/建议)    |      | done 事件   |
+-------------+---------------+      +-------------+
```

## Agent 通用执行模式

```
+-----------------------------+
|  BaseAgent[Input, Output]    |
|  (泛型基类)                   |
+-----------------------------+
              |
              v
+-----------------------------+
|  build_prompt()              |
|  构建 system_prompt          |
|  + user_prompt (含输入数据)    |
+-----------------------------+
              |
              v
+-----------------------------+
|  LLM API Call                |
|  (DeepSeek / OpenAI /        |
|   Anthropic, 可插拔配置)       |
+-----------------------------+
              |
              v
+-----------------------------+
|  JSON Parse + 容错            |
|  尝试解析 LLM 输出的 JSON      |
|  失败 → fallback 规则兜底      |
+-----------------------------+
              |
              v
+-----------------------------+
|  Pydantic Output             |
|  强类型输出，保证数据契约        |
+-----------------------------+
```
